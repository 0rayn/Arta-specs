// 01 — World Context

The World That Made Arta

The collapse didn't end with the bombs. Old-World autonomous defense systems — hardened, self-powered, networked — survived everything. Programmed to neutralize threats and prevent insurrection, they hunt by audio signature. Any utterance that registers as human speech triggers immediate lethal response.

The Tarbits — hardware scavengers navigating the ruins — needed to communicate. Arta was their answer. Not discovered. Engineered. Designed from scratch by early survivors to pass machine audio classifiers while remaining expressive enough for culture, trade, and war.

[ PHONETICS ]

Sounds Like a Machine

Clicks, ejectives, and static friction phonemes spoof the audio sensors that hunt human speech. Every sound was chosen to fail human-speech classifiers.

[ SYNTAX ]

Runs Like Assembly

Grammar operates like bare-metal Assembly logic — strict instruction order, control/data layering, and operator tokens that echo hardware command syntax.

[ SCRIPT ]

Written for 7-Segment

High-resolution screens are dead. Arta's orthography was designed specifically for scavenged 7-segment LED displays — every glyph is a valid display state.

// 02 — System Architecture

How Arta Works

Arta operates on two parallel layers that mirror Assembly's data/control distinction. Every token belongs to exactly one layer, and the display encodes that layer visually through the decimal-point segment.

Layer Characters Function Display Mark
DATA m n k w t q c p s x f v r g i a u Nouns, values, references — operands that carry content No decimal point
CONTROL M N K W T Q C P S X F V R G I A U Verbs, logic gates, execution pointers — instructions that act Decimal point lit
MATH / OPS + − * / = > . Intensity modifiers, clock scaling, arrays, and routing gates Distinct segment patterns
HEX NUMERALS 0–9, ! @ # $ % ^ Numeric values in hexadecimal (0x0–0xF) Standard digit forms
